## Null

Another primitive type used to represent nothing.

The practical difference between `null` and `undefined` is that `null` will never appear in your program by accident, `undefined` can appear in your program if you forgot to initialize a variable. If a variable stores `null` it was put there on purpose.

When you see `null` in a program it means that there is _supposed to be_ nothing, this is different than `undefined` which can happen by accident (this will become even more clear when you learn about objects in Debugging).

```js
console.log('-- null --');

// that's it, plain and simple
console.log(null);
```

<details>
<summary>🥚 typeof</summary>

```js
console.log('-- typeof null --');

// this is not intuitive, you just need to memorize it
console.log(null); // null
console.log(typeof null); // 'object'
```

</details>
<details>
<summary>🥚 strict equality</summary>

```js
console.log('-- null: strict equality --');

// null is only strictly equal to null
console.log(null === null); // true

// any other comparison is false
console.log(null === 'null'); // false
console.log(null === 0); // false
console.log(null === undefined); // false
```

</details>
<details>
<summary>🥚  strict inequality</summary>

```js
console.log('-- null: strict inequality --');
// this will always be the opposite of strict equality

console.log(null !== null); // false

// any other strict inequality with null will be true
console.log(null !== 'null'); // true
console.log(null !== 0); // true
console.log(null !== undefined); // true
```

</details>
<details>
<summary>🥚 variables with null</summary>

```js
console.log('-- variables with null --');

// you need to assign null to a variable, it does not happen by accident
let variable;
console.log(variable); // undefined
console.log(typeof variable); // 'undefined'

variable = null;
console.log(variable); // null
console.log(typeof variable); // 'object'

// you can also initialize a variable to null
let breakfast = null;
console.log(breakfast);
console.log(typeof breakfast);
```

</details>
